🔧 Correction Supabase
Le trigger de création de profil est absent. Va dans Supabase → SQL Editor et exécute ce SQL :
-- Fonction trigger CREATE OR REPLACE FUNCTION handle_new_user() RETURNS TRIGGER AS $$ BEGIN INSERT INTO public.profiles (id, display_name, role, favs, visit_counts) VALUES ( NEW.id, COALESCE(NEW.raw_user_meta_data->>'display_name', split_part(NEW.email,'@',1)), 'user', '{}', '{}' ) ON CONFLICT (id) DO NOTHING; RETURN NEW; END; $$ LANGUAGE plpgsql SECURITY DEFINER; -- Supprimer si existant puis recréer DROP TRIGGER IF EXISTS on_auth_user_created ON auth.users; CREATE TRIGGER on_auth_user_created AFTER INSERT ON auth.users FOR EACH ROW EXECUTE FUNCTION handle_new_user();
Après l'exécution, retente la création de compte. Si tu as déjà l'utilisateur dans Auth, va dans Table Editor → profiles et crée sa ligne manuellement.
